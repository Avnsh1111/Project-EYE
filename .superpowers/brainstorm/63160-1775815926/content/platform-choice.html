NativePHP Platform Strategy

NativePHP has two distinct products — pick the target platform(s) for the rebuild

A

NativePHP Mobile (iOS + Android)

Replaces the React Native app entirely. Uses Capacitor under the hood. Your Laravel/PHP code compiles into a native iOS/Android app. Auto-backup from camera roll, push notifications, background sync — all available via device APIs.

Single codebase PHP → Mobile Early Access — some rough edges iOS + Android
B

NativePHP Desktop (Mac + Windows + Linux)

Wraps the existing Laravel/Livewire web app into a desktop app using Electron. Adds system tray, native menus, folder watching for auto-import, background sync daemon, and native file dialogs. The web app runs inside a native window.

Stable + Production Ready Reuses existing Livewire UI Mac / Win / Linux
C

Both — Desktop App + Mobile App

Build a NativePHP Desktop app (wrapping the web UI) AND a NativePHP Mobile app (iOS/Android). The desktop app handles folder watching & auto-import from your computer. The mobile app handles camera roll backup from your phone. Both sync to the same self-hosted server.

Full Google Photos parity Most complex All platforms
Note on NativePHP Mobile

NativePHP Mobile (https://nativephp.com/docs/mobile) was announced and is in early access. It wraps your Laravel app with Capacitor to produce native iOS/Android binaries. Camera, GPS, push notifications, background tasks, and file system access are all available via PHP APIs.
